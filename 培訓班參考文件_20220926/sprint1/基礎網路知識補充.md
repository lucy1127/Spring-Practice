# 基礎網路知識補充

## 前言

在架設網站、網頁伺服器時，會需要查找沒有使用的 port，或是在確認其他網站、API 功能是否正常時，
也需要某些方式進行測試，上述過程中，我們可以直接運用 Windows 內建的功能下指令操作驗證。

## 課程目標

學習使用 ipconfig、telnet、netstat 指令，以及網路設備名詞的介紹。

作業練習：
嘗試對 google.com.tw 進行 telnet 指令，並觀察結果。

## 課程開始

### Windows 指令

- ipconfig：用來查找本地主機的網路資訊
  ![ipconfig](https://i.imgur.com/TgZ6qZn.png)

  - 電腦可能會有多張網卡，通常需要找的是 IPv4 位址
  - ipconfig 指令有許多可以附加的指令，可以下指令 ipconfig/command 查詢，請參考 [Mocrosift Docs](https://docs.microsoft.com/en-us/windows-server/administration/windows-commands/ipconfig)

- telnet：跟遠端的伺服器溝通
  Windows 系統通常不會直接開啟 telnet 功能，需要手動開啟，步驟如下 (Win 10 環境)：
  1. `Win + Q` 開啟搜尋列，或點擊工具列的放大鏡圖示，搜尋「Windows 功能」
    ![Windows 功能](https://i.imgur.com/dkSQNwE.png)
  
  2. 找到 Telnet Client 功能並勾選，按確定，並按步驟安裝
    ![Telnet Client](https://i.imgur.com/BOrm46C.png)

  3. 安裝完成後重新開啟 cmd，並輸入 `telnet {ip} {port}`
    ![telnet cmd](https://i.imgur.com/w4unxUz.png)
    ![telnet cmd](https://i.imgur.com/JVIF0zA.png)

- netstat：查看本地端網路連線狀況
  常用 netstat -anob：查看 port 被哪個應用程式（PID）占用，再透過工作管理員查看該 PID 的程式是什麼
  ![netstat](https://i.imgur.com/sXE2JBf.png)

### 名詞介紹

- 防火牆
  - 軟體：常見的就是 Windows 防火牆，或是防毒軟體，可以透過設定讓部分程式可以通透指定的 port、傳輸協定。
  - 硬體：同軟體防火牆，但目前有很多設備都已經被整合進同一台機器，包括內網網段切分、網段管理、傳輸控管等。
- Load Balance 負載平衡：
  - 在多台電腦、網路連接等資源使用中，透過資源分配的方式，以達到最佳化的運用。
  - 所以說很有可能是對外 IP 綁定在防火牆，再到 Load Balance，再傳遞到有空閒的 Server。
- 網段
  - 在 IPv4 中，我們通常會用來表示 IP 前三碼或前兩碼是否相同，同網段才有辦法互相連線。
- DHCP
  - 動態分配 IP，在網際網路設定中，若沒有手動指定 IP、DNS，通常都是由 DHCP 的方式取得 IP。
